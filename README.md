🎯 What is Revize?
Revize is an intelligent learning platform that helps you retain information effectively using scientifically-proven spaced repetition. Simply upload your study materials—whether it's a PDF document, images of your notes, or even handwritten diagrams—and our AI automatically creates comprehensive flashcards for you.

No more manually typing out flashcards. No more forgetting what you studied last week. Revize remembers when you need to review each concept and reminds you at the optimal time.

Perfect For:
Students preparing for exams
Professionals learning new skills
Researchers retaining complex information
Language Learners memorizing vocabulary
Anyone who wants to learn more effectively

🧠 Smart Spaced Repetition
Based on cognitive science research, Revize schedules your reviews at scientifically optimal intervals:

Day 1: First review (24 hours after learning)
Day 4: Second review (when you're about to forget)
Day 9, 16, 25, 39, 60: Subsequent reviews with increasing intervals
This pattern ensures information moves from short-term to long-term memory efficiently.

Manual Entry → Type notes directly
Documents    → Upload PDF or Word files

Our AI analyzes your content and automatically:

Identifies key concepts and important information
Extracts definitions, formulas, and explanations

Example:

If you upload a PDF about "Photosynthesis," the AI might create flashcards like:

Flashcard 1: "Light-Dependent Reactions" - Detailed explanation of how chloroplasts convert light to chemical energy
Flashcard 2: "Calvin Cycle Process" - Step-by-step breakdown of carbon fixation
Flashcard 3: "Photosynthesis vs Respiration" - Comparison and relationship between the two processes

Each flashcard follows a personalized review schedule:

    ↓

    ↓
Review 3 (Day 9)  ← Solidify retention
    ↓
... (Days 16, 25, 39, 60)

🧪 Why Spaced Repetition?
Spaced repetition is one of the most effective learning techniques, backed by decades of cognitive science research.

The Forgetting Curve
Without review, we forget 50% of new information within 24 hours and 90% within a week.

100% ┤ ╮
     │  ╲
 75% │   ╲_____ (with spaced repetition)
     │    ╲
 50% │     ╲
     │      ╲╲___
 25% │         ╲╲╲____
     │             ╲╲╲╲________
  0% └─────────────────────────────
     Day 0  1   4   9   16  25  39  60
How Revize Combats Forgetting
Revize reviews content just before you're about to forget it, strengthening memory each time:

Initial Learning (Day 0): New information is fragile
First Review (Day 1): Reinforces fresh memory
Second Review (Day 4): Moves info to medium-term memory
Subsequent Reviews (Days 9-60): Solidifies long-term retention
Result: Information retention of 80-90% even months later.

🚀 Local Development Setup
Prerequisites
Python 3.11+
Node.js 18+
PostgreSQL (optional, SQLite is default)
Google Gemini API Key
Backend Setup
# 1. Clone repository
git clone https://github.com/yourusername/revize_v2.git
cd revize_v2

# 2. Create virtual environment
python -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate

# 3. Install dependencies
pip install -r spaced-repetition/backend/requirements.txt

# 4. Setup environment variables
cd spaced-repetition/backend
cp .env.example .env
# Edit .env and add:
# GEMINI_API_KEY=your_gemini_api_key_here
# DATABASE_URL=postgresql://user:pass@localhost:5432/revize_db (optional)

# 5. Run migrations
python manage.py migrate

# 6. Create superuser (optional)
python manage.py createsuperuser

# 7. Run development server
python manage.py runserver
# Backend runs on http://localhost:8000
Frontend Setup
# 1. Navigate to frontend (in a new terminal)
cd spaced-repetition/frontend

# 2. Install dependencies
npm install

# 3. Run development server
npm run dev
# Frontend runs on http://localhost:5173
Access the Application
Open your browser and navigate to:

Frontend: http://localhost:5173

🐛 Common Issues & Solutions
GEMINI_API_KEY not found
# Solution: Add to backend/.env file
GEMINI_API_KEY=your-actual-api-key-here
CORS errors in development
# Solution: Check settings.py has:
CORS_ALLOWED_ORIGINS = [
    "http://localhost:5173",
]
Image upload fails
# Check:
# 1. File size < 5MB per image
# 2. File format is JPG/PNG/WEBP
# 3. Max 10 images at once
Database connection error
# For development, use default SQLite (no config needed)
# For production, ensure PostgreSQL is running and DATABASE_URL is correct
